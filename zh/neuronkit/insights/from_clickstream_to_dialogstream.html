Agentic 转型

从点击流到对话流：重塑 AI 时代的用户体验

新的 Web 范式

互联网正经历一次前所未有的转型：从静态页面与预定义路径，迈向由智能、主动的 AI 智能体驱动的动态生态。这不仅仅是“聊天机器人”，更是人与数字信息与服务互动方式的根本改变。

过去：页面的网络

🖱️

点击流

用户在链接的迷宫中穿梭，从被动的“文字袋子”中手动拉取信息。界面固定、旅程完全由用户驱动。

现在：智能体的网络

💬

对话流

用户与理解上下文、能预测需求并主动执行任务的智能体对话。体验从“操作”走向“协作”。

生成式 UI 的崛起

自然语言很强大，但不会完全取代图形界面。未来是对话与“动态生成 UI 组件”的无缝融合。智能体将按需实时生成表单/列表/按钮等界面，形成“结构化信息与非结构化语言流交织”的连续体验。

生成式 UI 协议：MCP‑UI 与 AG‑UI

要让生成式 UI 成为现实，Web 需要新的协议。MCP‑UI 与 AG‑UI 提供了让智能体生成并呈现动态界面的技术框架，衔接“智能体逻辑”与“可视化体验”。

MCP‑UI

为特定查询生成上下文化 UI 组件（如表单、列表、按钮）。更像服务器向客户端发送的一张“卡片”。

AG‑UI

代表“智能体即应用”的完整界面，是一个持续、统一的 UI，用于承载主动智能体、复杂任务与“人‑在‑回路”体验。

应用的 Agentic 演进

应用引入智能体将经历阶段性演进：从增强到主导，再到“智能体即应用”。这标志着从“用户操作软件”到“与智能系统协作完成任务”的转变。

打造基础：WebMCP

要让智能体高效工作，需要一个“智能体的 HTTP”。Web 多提供方计算协议（WebMCP）为此而生：让智能体通过结构化、机器可读的动作与服务交互，而不是解析可视 UI。

  • 转向结构化动作： 以稳定的“类 API 调用”替代脆弱的 UI 抓取，准确高效。
  • 范围化认证： 细粒度、安全地授权智能体代用户执行动作。
  • 结果导向激励： 从注意力经济（展示、点击）转向价值经济（任务完成）。
  • 标准化分发： 让智能体可发现、评估与选择最合适的工具/服务。

前行之路的核心挑战

要让 Web 真正“面向智能体”，必须解决当“机器而非人类”成为主要用户时的一些根本挑战——当前 Web 生态并非为此而设计。

准确性与效率

解析视觉布局（VLM）或 DOM 既慢又不可靠。需要转向结构化、语义化通信。

安全

如何在不暴露凭据、且不授予过宽权限的情况下，安全地让智能体代用户行动？

商业模式

在智能体场景中，基于注意力的模式（广告、点击）失效，需要基于计量访问或可验证结果的新模式。

分发

当“代码变得廉价”，智能体如何在海量重叠供应中发现与选择最佳工具？需要标准化机制。

“生成式 UI + 智能体协议”的融合，正在创造更直观、高效、协作的数字世界。

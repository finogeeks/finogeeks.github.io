Inference in the cloud; safe actions on the device. The app behaves like a neuron in a larger agentic system.

Context is Intelligence

Capture rich device-side signals and organize them into semantic, episodic, and procedural memories for truly relevant agent reasoning.

Secure On‑Device Actions

A capability model + policy engine ensures AI-triggered actions are vetted, consented, and auditable—before your app does anything.

Dialog Stream UX

ConvoUI blends natural language with embedded UI components so users can talk and tap their way through tasks.

What you get

  • Rich Context SDK: location, sensors, app telemetry, and more—streamed as scenario context.
  • On‑Device Sandbox: define features → capabilities → primitives; require consent for sensitive actions.
  • Cloud Memory Server: time-ordered logs promoted into semantic, episodic, and procedural memory.
  • Agent Routing: route intents to registered agents; receive action plans back.
  • ConvoUI: drop-in voice + chat with inline UI widgets (buttons, forms, product tiles).
  • Analytics & Audit: policy decisions, consents, and action trails for compliance.
  • Platform-agnostic: Android / iOS / IoT friendly; cloud-neutral.

Architecture at a glance

Device

Your app + NeuronKit SDK. Registers features, collects context, enforces actions via the on-device sandbox.

Cloud

Context Memory Server organizes logs into semantic/episodic/procedural memory and routes intents to agents.

Agents

Agents plan actions against your app’s registered capabilities; sandbox validates and executes safely.

How we differ from cloud sandboxes

Aspect Cloud Sandboxes (e.g., E2B/Modal) NeuronKit (On-Device)
Where tools run Server-side containers Directly in your app / device
Context richness Limited live device signals Full device context (with policy)
Privacy & consent Depends on cloud policies Policy engine + per-action consent on-device
Latency Round-trips for action execution Edge execution; lower perceived latency
UX Chat only, UI separate Chat + embedded UI components
